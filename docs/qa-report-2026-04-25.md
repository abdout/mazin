### Bug #1 🔴 CRITICAL — Hogwarts brand leak (title + manifest)

**Symptom**
- Every page in the running app shows `<title>هجورتس - نظام إدارة المدارس</title>` (Arabic) and `<title>Hogwarts - School Management System</title>` (English).
- `<meta name="description">منصة شاملة لإدارة المؤسسات التعليمية</meta>` ("Comprehensive platform for educational institutions").
- `/manifest.webmanifest` returns:
  ```json
  {
    "name": "Hogwarts School Management",
    "short_name": "Hogwarts",
    "description": "A comprehensive school automation school-dashboard...",
    "categories": ["education", "productivity", "business"],
    "shortcuts": [
      { "name": "Mark Attendance", "url": "/attendance" },
      { "name": "Scan QR Code", "url": "/attendance/qr-code" }
    ]
  }
  ```

**Root cause analysis**
- `siteConfig.name = "Abdout"` (`src/lib/site.ts:2`) and root layout uses `${siteConfig.name} - Port Sudan Logistics` for default title (`src/app/layout.tsx:8`). **Source is correct.**
- `grep -rn "Hogwarts" src` finds only code-comment references (`// Follows Hogwarts pattern`) — no user-facing strings in source.
- HTML response references compiled chunk `src_app_%5Blang%5D_(saas-marketing)_layout_tsx_09w7sui._.js` — but `(saas-marketing)` directory does **not exist** in `src/app/[lang]/`.
- ⇒ Stale Turbopack dev cache at `.next/dev` is serving artifacts compiled against a previous source state (likely a Hogwarts template fork that was renamed but not rebuilt).

**Fix**
```bash
# Stop dev server, then:
rm -rf .next .turbo
pnpm dev
```

**Acceptance**
- After rebuild, `curl -s http://localhost:3000/ar | grep -E '<title>'` returns Mazin/Abdout title.
- `curl -s http://localhost:3000/manifest.webmanifest` returns Port Sudan logistics manifest (or 404 — see Bug #2).

### Bug #3 🔴 CRITICAL — Public marketing routes redirect to login

**Symptom**
| URL | Expected | Actual |
|-----|----------|--------|
| `/ar/about` | Render about page | Redirects to `/ar/login?callbackUrl=%2Far%2Fabout` |
| `/ar/services` | Render services page | Redirects to `/ar/login?callbackUrl=...` |
| `/ar/track` | Render tracking entry | Redirects to `/ar/login?callbackUrl=...` |
| `/ar/track/TRK-NOTREAL` | Render not-found page | Redirects to `/ar/login?callbackUrl=...` |

**Reproduce**
```bash
curl -s -o /dev/null -w "%{http_code} %{redirect_url}\n" http://localhost:3000/ar/about
# 307 http://localhost:3000/ar/login?callbackUrl=%2Far%2Fabout
```

**Source analysis** — `src/routes.ts` does whitelist these:
- `publicRoutes` line 14, 20, 25 includes `/about`, `/en/about`, `/ar/about` ✓
- `publicRoutes` line 15, 21, 26 includes `/services`, `/en/services`, `/ar/services` ✓
- `publicRoutePrefixes` line 36, 37 includes `/en/track/`, `/ar/track/` ✓ (but **not** `/track` index — see Bug #4)

`src/middleware.ts:86-87` logic:
```ts
const isPublicRoute = publicRoutes.includes(pathnameWithoutLocale) ||
    publicRoutePrefixes.some(prefix => pathname.startsWith(prefix))
```
For `/ar/about`: `pathnameWithoutLocale = '/about'` → `publicRoutes.includes('/about')` = `true`. **Should NOT redirect.**

**Likely cause**
Same stale dev cache as Bug #1. The compiled middleware doesn't reflect the current `routes.ts`. Verify by:
1. Clean rebuild (Fix in Bug #1).
2. Re-run the same curl commands.

If the redirect persists post-rebuild, the actual bug is in middleware logic — possibly `pathnameWithoutLocale` is mis-stripping or `isAuthenticated` is returning truthy without a cookie.

**Acceptance**
- All four URLs above return HTTP 200 with correct page content for an unauthenticated visitor.

### Bug #4 🟡 HIGH — `/track` index page not whitelisted

`publicRoutePrefixes` lists `/en/track/`, `/ar/track/`, `/track/` (with trailing slash). This means:

**Fix** — add to `src/routes.ts:10-29`:

### Bug #6 🟡 — Form labels missing (WCAG 3.3.2 violation)

Every auth form uses placeholder-only inputs without `<Label>` elements. Screen readers announce "Edit text" with no field name. Verified at:
- `/ar/login` — name, password fields no `<Label>`

**Fix** — wrap each input with `<Label>`:
```tsx
<div className="space-y-2">
  <Label htmlFor="email">{dict.auth.login.email}</Label>
  <Input id="email" placeholder={...} {...field} />
</div>
```
